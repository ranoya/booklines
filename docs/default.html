Slidelines Scriba

!<6 I

@{O que é?} -+- O que é o Scriba

Scriba é um sistema de auto-layout, isto é, uma ferramenta que coloca automaticamente um documento em um tipo de layout específico. Ele é feito para formatar textos densos e estruturados, que são aqueles documentos predominantemente textuais que fazem uso intenso de outlines, como é teses, disserações e relatórios, onde encontramos tudo organizado através de tópicos (como 1.1, 1.2, 1.2.1, etc.).

Este sistema foi criado para respeitar esta rígida estruturação textual, criando espaços (e respiro) para que os tópicos (outlines) fiquem claramente visíveis, que listas ordenadas e não-ordenadas se encaixem perfeitamente no layout, que referências e textos paralelos possam ser apresentados, e que índices navegáveis possam orientar a navegação no documento.

Todos os elementos são inseridos utilizando pequenas notações/marcações no próximo texto, que orientam a ferramenta em como tratar cada uma das partes redigidas.

Estes elementos podem ser apenas distribuídos em regiões específicas do layout, ou podem ficar fixos na tela enquanto o scroll se desenvolve, e isto é definido pelas próprias marcações feitas.

Dois layouts são possíveis de serem utilizados: um onde há uma coluna de texto com espaço para referências à direita, e outro contendo duas colunas paralelas, que é importante quando há elementos como imagens, diagramas, tabelas, ou outras informações que precisam ser apresentadas em paralelo, fixadas ou não na tela enquanto o scroll se desenvolve.

Os textos podem ser lidos de arquivos online dos tipos markdown ou html, ou também podem ser construídos na plataforma de programação literária Observable. A utilização do Observable é recomendada para documentos parametrizados por programação, pois não se faz necessária nenhum tipo de tecnologia server-side para construção da interação. Mas para aqueles que preferirem utilizar arquivos markdown ou html, qualquer webserver que utilize PHP, Node, ASP, C#, e possa gerar documentos dinâmicos, pode ser utilizado para criar elementos programados ou cujos conteúdos sejam parametrizados.

!<5 II

@{Elementos} -+- Elementos

%< O layout com texto central e referências é composto por uma coluna à esquerda dedicada aos tópicos/outlines, uma coluna menor dedicada às referências, e uma coluna na lateral direita para os índices de navegação.

!>5

%< O layout com duas colunas (este que está sendo utilizado agora) também possui uma coluna à esquerda dedicada aos tópicos/outlines, duas colunas (menores que a coluna do layout único) para textos e elementos paralelos, e uma coluna na lateral direita para os índices de navegação. Nesta distribuição, não há espaço para as referências.
