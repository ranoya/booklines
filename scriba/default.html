@{O que é?} -+- O que é o Scriba

!<7 1.1

Scriba é um sistema de auto-layout, isto é, uma ferramenta que coloca automaticamente um documento em um tipo de layout específico. Ele é feito para formatar textos densos e estruturados, que são aqueles documentos predominantemente textuais que fazem uso intenso de outlines, como teses, disserações e relatórios, onde encontramos tudo organizado através de tópicos (como 1.1, 1.2, 1.2.1, etc.).

@> O documento html que contém
este texto está disponível em:
https://booklines.vercel.app/scriba/default.html

Ele está sendo formatado com o Scriba.

Este sistema foi criado para respeitar esta rígida estruturação textual, criando espaços (e respiro) para que os tópicos (outlines) fiquem claramente visíveis, que listas ordenadas e não-ordenadas se encaixem perfeitamente no layout, que referências e textos paralelos possam ser apresentados, e que índices navegáveis possam orientar a navegação no documento.

Todos os elementos são inseridos utilizando pequenas notações/marcações no próximo texto, que orientam a ferramenta em como tratar cada uma das partes redigidas.

Estes elementos podem ser apenas distribuídos em regiões específicas do layout, ou podem ficar fixos na tela enquanto o scroll se desenvolve, e isto é definido pelas próprias marcações feitas.

Dois layouts são possíveis de serem utilizados: um onde há uma coluna de texto com espaço para referências à direita, e outro contendo duas colunas paralelas, que é importante quando há elementos como imagens, diagramas, tabelas, ou outras informações que precisam ser apresentadas em paralelo, fixadas ou não na tela enquanto o scroll se desenvolve.

%<  
%>  

Os textos podem ser lidos de arquivos online dos tipos markdown ou html, ou também podem ser construídos na plataforma de programação literária Observable. A utilização do Observable é recomendada para documentos parametrizados por programação, pois não se faz necessário nenhum tipo de tecnologia server-side para construção da interação. Mas para aqueles que preferirem utilizar arquivos markdown ou html, qualquer webserver que utilize PHP, Node, ASP, C#, e possa gerar documentos dinâmicos, pode ser utilizado para criar elementos programados ou conteúdos parametrizados.

@< -+- II

@{Elementos} -+- Elementos

!<8 2.1

%< Os layouts são configurados para que respeitem:

!%>8  
%<  
  • O tamanho da tela, retirando os elementos complementares quando o dispositivo não oferece espaço suficiente;
  • Espaçamento para os elementos gráficos de listas ordenadas, ou listas não-ordenadas (como esta);
  • Uma navegação eficiente sobre tópicos e uma hierarquia forte estabelecida no documento.

%< O layout com texto central e referências é composto por uma coluna à esquerda dedicada aos tópicos/outlines, uma coluna menor dedicada às referências, e uma coluna na lateral direita para os índices de navegação.

%< O layout com duas colunas (este que está sendo utilizado agora e cuja imagem está ao lado) também possui uma coluna à esquerda dedicada aos tópicos/outlines, duas colunas (menores que a coluna do layout único) para textos e elementos paralelos, e uma coluna na lateral direita para os índices de navegação. Nesta distribuição, não há espaço para as referências.

%< Este tipo de layout não é recomendável para divisão do texto em duas colunas. Ao contrário da mídia impressa, cujo layout tem um tamanho fixo e fica completamente visível para o leitor, em um documento web ele se distribui de forma fluída pelo espaço disponível. Isto faz com que o cumprimento do texto seja sempre indefinido, e o obrigue a ficar subindo e descendo o scroll para dar continuidade na leitura, indo para o parágrafo seguinte que se encontra ao lado, mas cujo início está muito acima da posição em que ele se encontra na página.

%< A possibilidade de fixar elementos na tela enquanto o scroll se desenvolve é um recurso importante para que informações que exijam, por exemplo, uma referência gráfica ou um diagrama em paralelo, possam ser lidas com o próprio elemento presente. Esta é uma necessidade ergonômica importante quando algumas relações estão sendo apresentadas no documento.

%< Os elementos fixos podem permanecer ali até que sejam sobrepostos por outros elementos fixos, ou podemos estabelecer por quantos parágrafos (ou blocos) eles devem estacionar no scroll.

%< Cada tipo de elemento recebe uma marcação específica para que possa operar dentro do layout.

@< -+- IV

@{Uso} -+- Como Utilizar?

!<9 4.1

Para utilizar o Scriba você deve escrever seu texto no formato html ou markdown e disponibilizá-lo online através de uma URL em que o arquivo esteja acessível para leitura, ou escrevê-lo utilizando a plataforma Observable.. A segunda opção é recomendável para facilitar o processo (não há a necessidade de disponibilizar o arquivo através de algum webserver), além de oferecer a possibilidade de parametrizar a informação através de programação.

Se for utilizar um arquivo, basta incluir o link para este arquivo na URL do Scriba, no seguinte formato:

https://booklines.vercel.app/scriba/?url=[link https completo do arquivo]
@> Exemplo:
https://booklines.vercel.app/scriba/?url=https://booklines.vercel.app/scriba/default.html

Se for utilizar um notebook no Observable, o processo é bastante semelhante, mas você utilizará outra variável de url (a notebook), e informará o identificador do seu documento na plataforma que é composto pelo nome do usuário e o nome do arquivo, da seguinte forma:

https://booklines.vercel.app/scriba/?notebook=[@usuário/documento]
@> Exemplo:
https://booklines.vercel.app/scriba/?notebook=@ranoya/teste-scriba
